# 🧠 StudyMind - منصة المذاكرة والتحصيل الأكاديمي الذكية المتكاملة
### الدليل الشامل للتشغيل والإنتاج والنشر التجاري (Production-Ready Suite)

منصة **StudyMind** هي نظام كامل متكامل (Full-Stack) يعتمد على الذكاء الاصطناعي لمساعدة الطلاب على إدارة مسيرتهم الدراسية، توليد الاختبارات الذكية، تخليص المحاضرات، وتنظيم الوقت. تم تصميم هذه النسخة لتكون مهيأة بنسبة 100% للإطلاق التجاري المباشر.

---

## 🏗️ البنية البرمجية ونظام العمل (Architecture Overview)

تعتمد المنصة على بنية تحتية حديثة وموزعة لضمان أفضل مستويات الحماية والأداء العالي:
1. **الواجهة الأمامية (Client SPA):** واجهة سريعة وخفيفة مبنية بـ **React 18** و **Vite** مضافاً إليها جمالية التجاوب وحركات الانتقال الديناميكية عبر **Motion**.
2. **الخادم الخلفي (Backend API Node.js):** خادم خفيف وعالي الأداء ومحمي تماماً يعتمد على **Express.js** مع محرك تحويل وتجميع متكامل عبر **esbuild** لبيئات التشغيل.
3. **مستودع البيانات (PostgreSQL):** قاعدة بيانات سحابية علائقية لتسجيل الطلاب، والملفات المُحللة، والاختبارات، ومستويات التقدم.
4. **التخزين المؤقت والكاش (Redis Cache):** لإدارة الجلسات السريعة، وتخزين الـ Refresh Tokens، ومنع هجمات إغراق الطلبات (Rate Limiting).
5. **توليد الذكاء الاصطناعي (Gemini SDK):** محرك توليد الأسئلة، واستخراج محتوى الملفات والتحليل الذكي باستخدام أحدث نماذج الاستدعاء السحابي الآمن.
6. **بوابة الدفع الإلكتروني (Stripe Subscriptions):** لدعم الاشتراكات التلقائية المتكررة (باقة برو وباقة بريميوم).

---

## 🔒 نظام الحماية والتوثيق (JWT & Secure Authentication)

المنصة مجهزة بنظام توثيق وحماية فائق الأمان يعتمد على رموز الوصول السريعة والتحديث التلقائي:
* **Access Tokens (JWT):** يتم توليدها بصلاحية قصيرة (ساعة واحدة) وتوقيعها بمفتاح مشفر خاص لمنع تزوير الجلسات.
* **Refresh Tokens:** مشفرة بصلاحية طويلة (7 أيام) تُحفظ في قاعدة البيانات أو ذاكرة الكاش السريعة لتحديث الـ Access Token تلقائياً دون إزعاج المستخدم بطلب إعادة تسجيل الدخول بشكل متكرر.
* **Social Auth (Google & Apple):** مسارات جاهزة ومجهزة لاستقبال الـ ID Tokens والتحقق من صحتها وتوقيع المستخدم فورا في النظام.

## 🚀 التشغيل المحلي والتطوير (Local Development)

### 📋 المتطلبات الأساسية
* مثبت لديك **Node.js v20 أو أحدث**.
* قاعدة بيانات **PostgreSQL** نشطة (أو استخدام الذاكرة الافتراضية المدمجة للتجربة السريعة).

### 🛠️ خطوات التشغيل
1. قم بتثبيت الاعتمادات للبرنامج:
   ```bash
   npm install
   ```
2. انسخ ملف المتغيرات البيئية وقم بتحديث المفاتيح الخاصة بك:
   ```bash
   cp .env.example .env
   ```
3. ابدأ تشغيل خادم التطوير المشترك (الواجهة الأمامية والخلفية معاً بـ Hot-Reload):
   ```bash
   npm run dev
   ```
4. افتح المتصفح على الرابط التالي: [http://localhost:3000](http://localhost:3000).

## 🐳 النشر الفوري بـ Docker (One-Click Docker Setup)

المنصة مجهزة بالكامل للعمل في بيئة حاويات معزولة وآمنة بضغطة زر واحدة عبر **Docker Compose**:

### 📦 تشغيل النظام المتكامل (App + DB + Redis + Automated Backup)
كل ما عليك هو تشغيل الأمر التالي في المجلد الرئيسي للمشروع:
```bash
docker-compose up --build -d
```

### ⚙️ كيف يعمل نظام Docker Compose؟
يقوم بتهيئة وإنشاء أربع حاويات متصلة بشبكة داخلية معزولة:
1. **`studymind_app`:** تطبيق الويب الكامل (Node + Vite client) يعمل على المنفذ `3000`.
2. **`studymind_postgres`:** قاعدة بيانات PostgreSQL 16 لحفظ البيانات الحقيقية وتستخدم Volume مستقل لضمان عدم فقدان البيانات عند إعادة التشغيل.
3. **`studymind_redis`:** ذاكرة كاش سريعة ومحمية بكلمة مرور لإدارة الرموز والسرعة العالية.
4. **`studymind_db_backup`:** نظام نسخ احتياطي تلقائي مجدول، يأخذ نسخة احتياطية مشفرة ومضغوطة من قاعدة البيانات يومياً ويحفظها في مجلد `/backups` مع الإبقاء على آخر 7 نسخ فقط تلقائياً وتفريغ القديم لتوفير مساحة القرص.

## 🛠️ لوحة تتبع الأداء ومراقبة الأخطاء (Monitoring Telemetry)

تتضمن المنصة لوحة مراقبة حية ومبنية داخل لوحة التحكم لـ **Production Setup** تتيح للمشرفين:
* تتبع استهلاك المعالج والذاكرة الفعلي للـ Node.js Container.
* فحص حالة الاتصال بـ PostgreSQL وزمن الاستجابة (Latency in MS).
* تتبع سجل العمليات المباشر (Audit Logs) لمعرفة نجاح الدفع الإلكتروني وتوثيق المستخدمين.
* التحقق من سلامة مفاتيح الاتصال بخدمات Google Gemini و Stripe و AWS S3 بضغطة زر وتنبيه المشرفين حال سقوط أي منها.

## 📱 ربط التطبيق بـ Flutter (Mobile Application API)

الخلفية مهيأة بالكامل للعمل كـ **REST API Server** لتطبيق الهاتف المحمول (Flutter أو React Native):

### 🔑 أهم روابط المصادقة لتطبيق الجوال:
* **تسجيل مستخدم جديد:** `POST /api/auth/register`
* **تسجيل دخول:** `POST /api/auth/login`
* **تحديث الرموز:** `POST /api/auth/refresh`
* **بيانات المستخدم الحالية:** `GET /api/users/me` (مع إرسال الرمز في الـ Authorization Header كـ `Bearer <Token>`)

## 🔁 الأتمتة والنشر السحابي (CI/CD GitHub Actions)

الملف المرفق `.github/workflows/ci-cd.yml` يحتوي على مسار عمل متكامل يقوم بتنفيذ الآتي تلقائياً عند كل دفع كود فرعي أو رئيسي:
1. فحص الكود والتأكد من خلوه من الأخطاء التكوينية (Linter verification).
2. بناء حزم الواجهة والسرعة وعملية تجميع السيرفر التجاري.
3. بناء حاوية Docker للمشروع ورفعها إلى **Google Artifact Registry**.
4. نشر الحاوية فوراً إلى منصة **Google Cloud Run** كخدمة سحابية حية سريعة وقابلة للتوسع التلقائي اللامتناهي.
